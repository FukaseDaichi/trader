# 週次レポート仕様（構成・文体・LINE通知）

更新日: 2026-06-06 JST

レポートライター agent が、週次ファンダメンタル結果とテクニカル結果、直近 decision logs を Markdown にまとめます。出力は `reports/weekly_YYYY-MM-DD.md` と `reports/weekly_latest.md` です。

## 1. ゴール

- 今週の注目銘柄と理由を、業績とチャートの両面から説明する
- ユニバースの入替を要約する
- カジュアルだが、数値・日付・出典は入力 JSON に忠実に扱う
- 投資助言ではない旨の免責を必ず入れる

## 2. 女の子ナビ・ペルソナ

| 項目 | 内容 |
|---|---|
| 名前 | 既定「あおい」。`settings.curation.report.persona_name` で変更可 |
| 一人称 | 「わたし」 |
| 読者 | 「キミ」 |
| 語尾 | 「〜だね！」「〜だよ！」「〜なんだ〜」「〜かも！」 |
| 絵文字 | 1 セクション 1〜2 個程度 |
| 色 | 日本式: 赤=上昇 / 青=下落 |

禁止:

- 「絶対上がる」「必ず儲かる」などの保証
- 入力 JSON にない数値、日付、銘柄、出典の捏造
- カジュアルさを理由に事実を省略・改変すること

## 4. LINE通知

`scripts/curation_notify.py` が週次 workflow の最後に実行されます。

現行仕様:

- `--report reports/weekly_<DATE>.md` を受け取る
- `TRADER_REPO_SLUG`、または `git remote.origin.url` から GitHub URL を組み立てる
- URL は `https://github.com/<owner>/<repo>/blob/<branch>/reports/weekly_<DATE>.md`
- branch は `TRADER_REPORT_BRANCH` 未設定時 `main`
- レポート本文の最初の `###` 見出しを headline として通知文に入れる
- LINE 設定がない場合は送信せず、送信予定本文を標準出力に出す
- 送信失敗は非致命

通知文の形:

```text
あおいだよ！今週の日本株レポートができたよ〜！📈✨
今週の注目は <headline> だね！
続きはこちら👇
https://github.com/...
（投資は自己責任だよ！最後は自分で決めてね🙆‍♀️）
```

## 5. 現行制約

- Markdown の front matter、免責、銘柄コード存在チェックを機械的に検証する validator は未実装
- LINE 通知にはリトライがない
- レポートは GitHub blob URL として通知され、ダッシュボード UI には表示されない
